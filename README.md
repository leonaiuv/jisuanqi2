# 抖音电商投放计算器（Web）

一个面向运营/投放协同的 PC 看板小工具：**同屏计算“运营口径（今日累计）”与“投放口径（平台近 1 小时退款率）”**，并支持目标费率换算与本地方案保存对比。

## 功能

- **实时指标（运营口径｜今日累计）**
  - 输入：成交金额、广告消耗、今日实时退款率
  - 输出：退款前 ROI、实时费率、净 ROI、今日净成交金额
- **投放端对齐（平台近 1 小时退款率）**
  - 默认：用“今日成交/消耗 + 近 1 小时退款率”做**估算**
  - 高级（可选）：填写近 1 小时成交/消耗，做到成交/消耗/退款率**同窗口精算**
- **目标换算（目标费率 → 预期 ROI）**
  - 输入：目标费率、退款率
  - 输出：目标净 ROI、预期退款前 ROI（退款前）
- **月度费率（本月累计）**
  - 输入：本月成交、本月消耗、本月退款率（已发生）
  - 输出：月度费率、月度净 ROI、月度退款前 ROI
  - 高级（可选）：输入“预计最终退款率”，用于月末预估（解决退款延后导致的低估）
- **方案保存（可选）**
  - 仅存本地浏览器（`localStorage`），可保存多个输入组合并一键回填

## 公式（口径与页面一致）

- **退款前 ROI（运营常看）**
  - \(ROI_{gross} = 成交金额 / 广告消耗\)
- **净 ROI（剔除退款后，投放/费率对应的 ROI）**
  - \(ROI_{net} = 成交金额 \* (1-退款率) / 广告消耗\)
- **费率（你习惯用来调价的口径）**
  - \(费率 = 广告消耗 / (成交金额 \* (1-退款率))\)
  - 结论：**费率 = 1 / 净 ROI**

## 输入约定

- **金额**：单位元，支持 `,` 分隔（如 `1,250,000`）
- **百分比/费率输入**：支持 `10%`、`10`、`0.1`
  - `10%` / `10` 会按 **10%** 处理
  - `0.1` 会按 **0.1（即 10%）** 处理（兼容小数写法）
- **退款率范围**：0%~100%
- **目标费率**：允许大于 100%（例如 150%）
- **未填写退款率**：页面会提示并默认按 **0%** 计算（“预计最终退款率”属于可选预测项，不填则不计算预测结果）

## 月度退款率口径建议

- **推荐默认（对账/当前净产出）**：用“本月累计已发生退款率”  
  - 即：**截至今天**，本月已退款金额 / 本月成交金额
  - 优点：口径清晰，和“当前净成交/当前净 ROI”一致
  - 注意：由于退款会延后，这个值通常**低估最终退款率**
- **如果你要做月末预估/目标管理**：用“预计最终退款率（成熟退款率）”
  - 可取：历史同类活动/同品类的成熟退款率，或按成交日归因的 D+7 / D+14 退款率
- **不建议**把“每天的下播实时退款率”简单做算术平均当作月度退款率  
  - 不同天成交规模不同会被平均偏置；且新近成交的退款天然更滞后，会进一步低估

## 本地运行

安装依赖：

```bash
npm install
```

启动开发：

```bash
npm run dev
```

构建生产包：

```bash
npm run build
```

本地预览生产包：

```bash
npm run preview
```
