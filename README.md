# 抖音电商投放计算器（Web）

一个面向运营/投放协同的 PC 看板小工具：**同屏计算“运营口径（今日累计）”与“投放口径（平台近 1 小时退款率）”**，并支持目标费率换算与本地方案保存对比。

## 功能

- **实时指标（运营口径｜今日累计）**
  - 输入：成交金额、广告消耗、今日实时退款率
  - 输出：退款前 ROI、实时费率、净 ROI、今日净成交金额
- **投放端对齐（平台近 1 小时退款率）**
  - 默认：用“今日成交/消耗 + 近 1 小时退款率”做**估算**
  - 高级（可选）：填写近 1 小时成交/消耗，做到成交/消耗/退款率**同窗口精算**
- **目标换算（目标费率 → 预期 ROI）**
  - 输入：目标费率、退款率
  - 输出：目标净 ROI、预期退款前 ROI（退款前）
- **方案保存（可选）**
  - 仅存本地浏览器（`localStorage`），可保存多个输入组合并一键回填

## 公式（口径与页面一致）

- **退款前 ROI（运营常看）**
  - \(ROI_{gross} = 成交金额 / 广告消耗\)
- **净 ROI（剔除退款后，投放/费率对应的 ROI）**
  - \(ROI_{net} = 成交金额 \* (1-退款率) / 广告消耗\)
- **费率（你习惯用来调价的口径）**
  - \(费率 = 广告消耗 / (成交金额 \* (1-退款率))\)
  - 结论：**费率 = 1 / 净 ROI**

## 输入约定

- **金额**：单位元，支持 `,` 分隔（如 `1,250,000`）
- **百分比/费率输入**：支持 `10%`、`10`、`0.1`
  - `10%` / `10` 会按 **10%** 处理
  - `0.1` 会按 **0.1（即 10%）** 处理（兼容小数写法）
- **退款率范围**：0%~100%
- **目标费率**：允许大于 100%（例如 150%）
- **未填写退款率**：页面会提示并默认按 **0%** 计算

## 本地运行

安装依赖：

```bash
npm install
```

启动开发：

```bash
npm run dev
```

构建生产包：

```bash
npm run build
```

本地预览生产包：

```bash
npm run preview
```
